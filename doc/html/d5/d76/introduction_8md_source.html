1 # Introduction
2 
3  One man\'s constant is another man\'s variable. -- A.J. Perlis
4 
5 For humans extracting information from reading the web content is not very challenging,
6 but for machine it's exponentially more complex. To resolve this complexion in extraction
7 important and relevant information from the web the extension of WWW was proposed.
8 This extension is called `semantic web`.
9 
10 It helps with extracting and processing information from web by human but also more importantly to machines.
11 
12 Probably most important goal of the semantic web is to describe relationships between
13 individual data sources. For this exact purpose as one of the pillars of the semantic
14 web `Resource Description Framework` - `(RDF)` exists. The RDF denotes those
15 relationships as `triples` which then form vast graphs.
16 
17 Primary format for description of triples in RDF become `XML/RDF`. Due to it's markup
18 based syntax of XML it's easy for the machines to extrapolate the data from it using
19 already available tools and libraries. The cons of this format are huge file sizes
20 because of the markup overhead and the XML syntax is not optimized for human
21 readability. This predicament gave birth to other formats how to improve the
22 description of the triples. Those formats are usually not compatible between each
23 other and present different solution to the problems.
24 
25 One of the most know formats is `RDF/Turtle` which is a subset of larger and more
26 powerful syntax Notation-3 usually denoted as simply `N3`.
27 
28 This thesis focuses exactly on this format and it's simpler subset `N-Triples`.
29 The goal of the implementation if to create memory-efficient, high performance parser
30 to allow fast readout of the triples from files.
