1 #1 Resource Description Framework
2 
3 Resource Description Framework (from here on referred to as simply `RDF`)
4 is `W3C` specification for description and modeling of objects, their
5 relationships and properties. Data stored in RDF format is formed by
6 `TRIPLET: SUBJECT - PREDICAMENT - OBJECT`. The triplet tells us:
7 
8  "SUBJECT has property PREDICAMENT and the predicament has value of
9  OBJECT"
10 
11 For storing RDF data the special storage techniques and tools were
12 designed, which allows additional functions such as querying, inserting or
13 deleting the data inside.
14 
15 The other rather interesting property of RDF is ability to deduce
16 additional information - in simplicity it is creation of triplets not
17 (physically) existing in the RDF storage.
18 
19 ##1.1 Relationship with the Semantic Web
20 
21 Semantic web is all about the data. On the internet there are large
22 quantities of the data available in different forms. The most dominant
23 forms are still texts, images and tables. Those are easy to understand for
24 human, as he draws his own logical conclusions about the data and links
25 them together.
26 
27 Machines do not possess such ability to do logical deductions, so how it
28 can then recognize relationships between information? This exact problem
29 is challenged by `Semantic Web`.
30 
31 The basic principle is to complement the in human readable form (text,
32 image ...) by metadata. And from the metadata the machine will determine
33 relationships between objects.
34 
35 The semantic web development is driven by the World Wide Web Consortium
36 and it's technologies specifications contains specification for `RDF` and
37 `OWL`.
38 
39 ##1.2 RDF as a graph
40 At the data represented in `RDF metadata` we can look as on oriented
41 graph. The root node is represented by subject, edge is predicament and
42 the result node is object. The nodes can be of following types:
43  * Entities (has its own definition in dictionary, has its own URI)
44  * Literals
45  * Blank nodes (not identified by URI)

47 ##1.3 RDF serialization formats
48 
49 For exchanging the `RDF metadata` there are numerous file formats in use.
50 The main focus of this thesis in on the `Notation 3` and particularly
51 it's subset: `Turtle`, `N-Triples` and `N-Quads`. Other well known formats
52 are: `RDF/XML`, `RDF/JSON`, `TriG`. For the following discussion about
53 each format the following example metadata will be used:
54 
55 | OBJECT | PERDICAMENT | SUBJECT |
56 | -- | -- | -- |
57 |ex:schools#VSB |ex:studiesAt | ex:Student |
58 |"xyz123" | ex:login | ex:Student |
59 | ex:Student | ex:teaches | ex:Profesor |
60 
61 ###1.3.1 RDF/XML
62 Primary format for the data specified in `W3C` documents in 1999 for semantic web RDF description.
63 ```xml
64 <?xml version="1.0" encoding="UTF-8"?>
65 <rdf:RDF
66  xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
67  xmlns:ex="http://example.com/#">
68 
69  <rdf:Description rdf:about="http://example.com/Student">
70  <ex:studiesAt>http://example.com/schools#VSB</ex:studiesAt>
71  <ex:login>xyz123</ex:login>
72  </rdf:Description>
73 
74  <rdf:Description rdf:about="http://example.com/Profesor">
75  <ex:teaches>http://example.com/Student</ex:teaches>
76  </rdf:Description>
77 </rdf:RDF>
78 ```

80 ###1.3.2 RDF/JSON
81 Based on format which drives `Web 2.0`. `JSON` notation was specified for
82 javaScript objects ( hence the name JSON - JavaScript Object Notation). It
83 gained popularity in recent years and therefore it should not be supprise
84 to anyone that it's used as alternative to more verbose `RDF/XML` syntax
85 
86 ```json
87 {
88  "http://example.org/Student" : {
89  "http://example.org/#studiesAt" :
90  [
91  {
92  "value" : "http://example.com/schools#VSB",
93  "type" : "uri"
94  }
95  ],
96  "http://example.org/#login" :
97  [
98  {
99  "value" : "xyz123",
100  "type" : "literal"
101  }
102  ]
103  },
104  "http://example.org/Profesor" : {
105  "http://example.org/#teaches" :
106  [
107  {
108  "value" : "http://example.com/Student",
109  "type" : "uri"
110  }
111  ]
112  }
113 }
114 ```

116 ###1.3.3 Turtle
117 Subset of the `Notation 3` syntax omitting some less common construction
118 such as implication (`=>`) and keywords such as `has`. Currently most
119 common alternative to the `RDF/XML` syntax.
120 
121 ```ruby
122 @prefix ns0: <http://example.com/#> .
123 
124 <http://example.com/Student>
125  ns0:studiesAt "http://example.com/schools#VSB" ;
126  ns0:login "xyz123" .
127 
128 <http://example.com/Profesor> ns0:teaches "http://example.com/Student" .
129 ```
130 
131 ###1.3.3 N-Triples
132 The subset of the `Turtle` notation simplifying it's syntax and omitting
133 more complex construction in favor of very simple triple syntax.
134 
135 ```xml
136 <http://example.com/Student> <http://example.com/#studiesAt> "http://example.com/schools#VSB" .
137 <http://example.com/Student> <http://example.com/#login> "xyz123" .
138 <http://example.com/Profesor> <http://example.com/#teaches> "http://example.com/Student" .
139 ```
140 
141 As is seen from those examples, the `N3` based examples are much easier to
142 read to human.
