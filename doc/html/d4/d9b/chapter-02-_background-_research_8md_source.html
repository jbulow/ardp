5 ## 2.1 Resource Description Framework
6 
7 Resource Description Framework (from here on referred to as simply `RDF`) is `W3C`
8 specification for description and modeling of objects, their relationships and
9 properties. Data stored in RDF format is formed by
10 `TRIPLET: SUBJECT - PREDICAMENT - OBJECT`. The triplet tells us:
11 
12  "SUBJECT has property PREDICAMENT and the predicament has value of OBJECT"
13 
14 For storing RDF data the special storage techniques and tools were designed, which
15 allows additional functions such as querying, inserting or deleting the data inside.
16 
17 The other rather interesting property of RDF is ability to deduce additional
18 information - in simplicity it is creation of triplets not (physically) existing
19 in the RDF storage.

21 ### 2.1.1 Relationship with the Semantic Web
22 
23 Semantic web is all about the data. On the internet there are large quantities of the
24 data available in different forms. The most dominant forms are still texts, images
25 and tables. Those are easy to understand for human, as he draws his own logical
26 conclusions about the data and links them together.
27 
28 Machines do not possess such ability to do logical deductions, so how it can then
29 recognize relationships between information? This exact problem is challenged
30 by `Semantic Web`.
31 
32 The basic principle is to complement the in human readable form (text, image ...)
33 by metadata. And from the metadata the machine will determine relationships
34 between objects.
35 
36 The semantic web development is driven by the World Wide Web Consortium
37 and it's technologies specifications contains specification for `RDF` and `OWL`.
38 
39 ###2.1.2 RDF as a graph
40 At the data represented in `RDF metadata` we can look as on oriented graph.
41 The root node is represented by subject, edge is predicament and the result
42 node is object.
43 The nodes can be of following types:
44  * Entities (has its own definition in dictionary, has its own URI)
45  * Literals
46  * Blank nodes (not identified by URI)

48 ###2.1.3 RDF serialization formats
49 
50 For exchanging the `RDF metadata` there are numerous file formats in use. The main
51 focus of this thesis in on the `Notation 3` and particularly it's subset: `Turtle`,
52 `N-Triples` and `N-Quads`. Other well known formats are: `RDF/XML`, `RDF/JSON`, `TriG`.
53 For the following discussion about each format the following example metadata will be used:
54 
55 | OBJECT | PERDICAMENT | SUBJECT |
56 | --- | --- | --- |
57 |ex:schools#VSB |ex:studiesAt | ex:Student |
58 |"xyz123" | ex:login | ex:Student |
59 | ex:Student | ex:teaches | ex:Profesor |

61 ###1.3.1 RDF/XML
62 Primary format for the data specified in `W3C` documents in 1999 for semantic web RDF description.
63 ```xml
64 <?xml version="1.0" encoding="UTF-8"?>
65 <rdf:RDF
66  xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
67  xmlns:ex="http://example.com/#">
68 
69  <rdf:Description rdf:about="http://example.com/Student">
70  <ex:studiesAt>http://example.com/schools#VSB</ex:studiesAt>
71  <ex:login>xyz123</ex:login>
72  </rdf:Description>
73 
74  <rdf:Description rdf:about="http://example.com/Profesor">
75  <ex:teaches>http://example.com/Student</ex:teaches>
76  </rdf:Description>
77 </rdf:RDF>
78 ```

80 ###1.3.2 RDF/JSON
81 Based on format which drives `Web 2.0`. `JSON` notation was specified for javaScript
82 objects ( hence the name JSON - JavaScript Object Notation). It gained popularity in
83 recent years and therefore it should not be supprise to anyone that it's used
84 as alternative to more verbose `RDF/XML` syntax
85 
86 ```json
87 {
88  "http://example.org/Student" : {
89  "http://example.org/#studiesAt" :
90  [
91  {
92  "value" : "http://example.com/schools#VSB",
93  "type" : "uri"
94  }
95  ],
96  "http://example.org/#login" :
97  [
98  {
99  "value" : "xyz123",
100  "type" : "literal"
101  }
102  ]
103  },
104  "http://example.org/Profesor" : {
105  "http://example.org/#teaches" :
106  [
107  {
108  "value" : "http://example.com/Student",
109  "type" : "uri"
110  }
111  ]
112  }
113 }
114 ```

116 ###1.3.3 Turtle
117 Subset of the `Notation 3` syntax omitting some less common construction such as
118 implication (`=>`) and keywords such as `has`. Currently most common alternative to
119 the `RDF/XML` syntax.
120 
121 ```ruby
122 @prefix ns0: <http://example.com/#> .
123 
124 <http://example.com/Student>
125  ns0:studiesAt "http://example.com/schools#VSB" ;
126  ns0:login "xyz123" .
127 
128 <http://example.com/Profesor> ns0:teaches "http://example.com/Student" .
129 ```
130 
131 ###1.3.3 N-Triples
132 The subset of the `Turtle` notation simplifying it's syntax and omitting more complex
133 construction in favor of very simple triple syntax.
134 
135 ```xml
136 <http://example.com/Student> <http://example.com/#studiesAt> "http://example.com/schools#VSB" .
137 <http://example.com/Student> <http://example.com/#login> "xyz123" .
138 <http://example.com/Profesor> <http://example.com/#teaches> "http://example.com/Student" .
139 ```

152 ### 2.2.1.1 Regular expressions
153 
154 > ##### Definition 2.1:
155 > A **string** *s* over an alphabet **Σ** is finite sequence of symbols drawn from that alphabet. The **length** of a string s is the number of symbols in *s*. The **empty string**, denoted *ε*, is a special string of length zero.
156 
157 
158 > ##### Definition 2.2 :
159 > A language *L* over alphabet **Σ** is a set of strings over this alphabet.
160 >
161 > **NOTE:**_The empty set and the set **{ε}** that contains only the empty string are languages under this definition_
162 
163 Less formal interpretation of the definitions states that: *string* is sequence of characters and *language* is set of strings. The most common operations on languages are union (L∪M), concatenation (LM) and Kleene closure (L<sup>∗</sup>).
