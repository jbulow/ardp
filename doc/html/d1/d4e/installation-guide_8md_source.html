1 #Installation guide
2 Following chapters outlines the necessary dependencies and environment for building the **ARDP**. The tested environment were OSX and Ubuntu, but generic approach should apply to all *nix platforms.
3 
4 _The recommened approach on GNU/Linux is to translate the Debian apt-get command into the package manager command of chosen distribution, as package managers usually resolve additional decencies which may be required for the packages and were not discovered during their presence on test distribution._
5 
6 ##Linux dependencies
7 The generic linux installation requires several tools to be installed. This guide suppose that the distribution has the basic development support and that _llvm-clang_ is already installed and is relatively up-to-date (3.5.0+). If the target platform is Debian based distribution, single command should build and install all required dependencies:
8 ```sh
9  sudo apt-get -y install libdispatch-dev
10  zlib1g-dev bzip2 libbz2-dev autotools-dev
11  autoconf ragel lemon
12 ```
13 Zlib installation:
14 ```sh
15  wget http://www.zlib.net/zlib-1.2.8.tar.gz
16  tar -xvzf zlib-1.2.8.tar.gz
17  cd zlib-1.2.8
18  ./configure && make && sudo make install
19 ```
20 
21 BZip2 installation:
22 ```sh
23  wget http://www.bzip.org/1.0.6/bzip2-1.0.6.tar.gz
24  tar -xvzf bzip2-1.0.6.tar.gz
25  cd bzip2-1.0.6
26  ./configure && make && sudo make install
27 ```
28 
29 LibDispatch installation:
30 ```sh
31  #Needs to have CLANG installed for the BlockRuntime
32  git clone git://git.macosforge.org/libdispatch.git
33  cd libdispatch
34  sh autogen.sh
35  ./configure CC=clang --with-blocks-runtime=/usr/local/lib
36  make check
37  make && sudo make install
38 ```

40 ##OSX dependencies
41 In order to install the dependencies used to build the ARDP on OSX, one should install XCode using the the AppStore. After installation of the Xcode, command for installing the CLT should be entered in terminal:
42 
43 ```sh
44  xcode-select --install
45 ```
46 If any of the dependencies are missing, it is recommended to install Homebrew package manager from **https://brew.sh** and installing the depencecies from it, or installing **wget* command and follow the steps as specified in linux generic installation from source.
47 
48 ##Installing the ARDP
49 After all dependencies are installed the ARDP should be ready to be build and installed.
50 
51 ARDP can be obtained using two methods: one is the ARDP version included on CD which is part of this thesis. In such case one should copy the **_ardp-*.tar.gz_**, unpack it and skip the following steps related to git.
52 
53 The other method is to download the sources from github git repository. The github has the newest version of the work regardless of the status of CD and is recommended as source, for improvements and changes are likely to be made to codebase. The github code which is identical to the content of the CD is tagged with label 'thesis'.
54 
55  ```sh
56  git clone https://www.github.com/michto01/ardp
57  cd ardp
58  sh autogen.sh # bootstrap the package.
59  ```
60 
61 As is visible in the previous listing, the git repository requires aditional step, namely running the script autogen.sh, which creates files, which are target specific and therefore are in .gitignore file.\bigbreak
62 
63 Optionally, one can generate the new code from the tools ragel and lemon. This allows to change and fine-tune the behavior of the generators tools; eg. change the style of generated output for ragel:
64 
65 ```sh
66  # from ardp root directory
67  ragel src/lib/lexers/turtle.rl -G2 src/lib/lexers/turtle.c
68  cd src/lib/parsers
69  lemon -s ./turtle_parser.y
70 ```
71 
72 The last step is to finally build the ARDP. It is done using the standard Unix autotools combo; to maintain the clarity of the package, build is done in separate directory as to avoid processed files mixup with source code:
73 
74 ```sh
75  mkdir build && cd build # optional
76  ../configure # ./configure if building from root
77  make
78  make check # optional
79  sudo make install
80 ```
