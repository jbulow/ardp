1 # GNU Make toolchain
2 
3 Most of the open-source project use the build tools to help people to access
4 the program easier and to unify the building process.
5 
6 One of such tools is the GNU Autotools toolchain. It consists of several programs
7 such as `autoconf`, `aclocal` and `make`.
8 
9 This toolchain helps to automate creation of Makefiles which then compile programs
10 and it`s dependencies.
11 
12 The normal working process for building process is then simplified for the user.
13 It usually consist of four steps:
14 ```
15  ./configure [-optional-paramaters]
16  make
17  make check
18  make install
19 ```
20 
21 In first step we call the script which then populates all of the Makefile templates
22 in our project and check for all dependencies and requirements.
23 
24 If all of the requirements are met then we can simply follow the next command
25 `make` which calls the tools such as compilers and preprocessors to compile
26 the project. This step takes place in the build directory if the prefix was not
27 specified.
28 
29 The next (optional) step is to run self test on the compiled binaries and libraries.
30 
31 If all is well we then proceed to the final step of installing the necessery binaries
32 and libraries into the host system. In this step the `sudo` (or equivalent) is
33 required because the application may be copied to locations requiring administator
34 priviledges to modify.
